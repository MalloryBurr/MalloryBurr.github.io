Debugging Methods

Include a summary of the debugging methods from Chapter 4 that you incorporated into the form in your website:


I used several different methods to help me debug my code as I completed the project. One of the main methods was using alerts and simply printing things to the screen. This helps me know whether the program is entering certain functions and blocks of code. I also found it very helpful for being a more obvious way of validating the processing of input. I could validate that my input was correctly taken in, stored, and processed. It was especially helpful for validating my logic on functions and if/else statements. Sometimes with several if statements, it can get confusing just what the code needs to do, and alerts allow me to see when the code is doing what I need it to do.

Another big assistance with debugging was NotePad++ itself. There were times I would forget semicolons and curly braces, and it would flag them in a different color. This helped quickly point out my syntax errors, and keep me more organized. I also used the validator tool on w3schools to help catch any other syntax errors. I try to read through my code before turning to the validator, but an extra run through is always helpful. Even if my code is running correctly, there could still be errors.
